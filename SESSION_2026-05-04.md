# Session 2026-05-04 — Audio-Stutter Fix + Chrome-Trigger Block

## Was funktioniert (verifiziert via AUTO_SKIP_TEST am Vormittag)

**Audio-Stutter ELIMINIERT** über zwei Fixes (siehe POSTMORTEMS.md für Details):

1. **I2S DMA ISR Priorität** in `m5stack_tab5/m5stack_tab5.c:bsp_audio_init`:
   - `chan_cfg.dma_desc_num = 16` (default 6 → 85ms DMA-Headroom)
   - `chan_cfg.intr_priority = 5` (default 0/LOWMED → I2S-ISR wird nicht mehr von WiFi/anderen ISRs > 100ms verzögert)
   - **Effekt:** I2S_SLOW 22→2, AUDIO_GLITCH 10→1, worst write_ms 537→55

2. **`std::regex` Stack-Overflow Crash** in `cspot/bell/CMakeLists.txt:31`:
   - `option(BELL_DISABLE_REGEX ... ON)` (default OFF)
   - `cspot/bell/main/io/URLParser.cpp` braucht zusätzlich `<cstring>` include für die Manual-Parse-Fallback
   - **Effekt:** kein Stack-Protection-Fault mehr in `cdn_dl` task bei Spotify-CDN-URLs

3. **CDNAudioFile Optimierungen** (`cspot/include/CDNAudioFile.h`, `cspot/src/CDNAudioFile.cpp`):
   - `STREAM_START_THRESHOLD = 96 * 1024` (war 128KB; verifiziert via `_maxReadNeed` Sentinel)
   - cdn_dl Task: priority 3, pinned to core 0 (entlastet core 1 wo cspot_player läuft)
   - Abort guard für Skip-during-Prefetch (verhindert callback während dlAbort)

4. **Buffer-Sentinels** (`cspot_idf/CSpotTask.cpp`):
   - `[BUF_FILL]` (jede 256 Calls)
   - `[BUF_EMPTY]` / `[BUF_REFILL]` (Buffer-Drain Detection)
   - 1 MB Circular-Buffer (war 512 KB)

5. **AUDIO_START Sentinel** (`Tab5AudioSink.cpp`): erste feedPCMFrames-Call nach setParams loggt `[AUDIO_START]` für Skip-zu-Audio Latenzmessung

6. **`I2S_SLOW` Sentinel** (`Tab5AudioSink.cpp`): warnt wenn `i2s_write` > 30ms blockiert
